Host Your Calendar Website

Your Historical Events Calendar is a static website built with HTML, CSS, and JavaScript. This means it can be hosted on many free platforms without requiring a server or database.

Note: We've created a deployment script deploy-to-github.bat in your project folder to make GitHub Pages deployment easier.

Hosting Options

1. GitHub Pages (Recommended)

GitHub Pages is a free hosting service that takes HTML, CSS, and JavaScript files straight from a repository on GitHub.

  • Create a GitHub account at github.com
  • Create a new repository (e.g., "historical-events-calendar")
  • Upload all your project files to the repository
  • Go to repository Settings → Pages
  • Select "main" branch as the source and click Save
  • Your site will be published at https://yourusername.github.io/repository-name/

Quick Deploy: Run the deploy-to-github.bat script in your project folder for guided deployment.

2. Netlify

Netlify offers free hosting with a simple drag-and-drop interface.

  • Go to netlify.com and create an account
  • Click "New site from Git" or drag your project folder to the dashboard
  • Follow the prompts to deploy your site
  • Your site will be available at your-site-name.netlify.app

3. Vercel

Vercel provides fast and free hosting for static websites.

  • Go to vercel.com and create an account
  • Click "New Project" and import your repository or upload files
  • Configure project settings (not needed for this static site)
  • Click "Deploy" and your site will be live at your-site-name.vercel.app

4. Cloudflare Pages

Cloudflare Pages offers fast, secure hosting with global CDN.

  • Create an account at cloudflare.com
  • Go to the Pages section and click "Create a project"
  • Connect to your GitHub/GitLab account and select your repository
  • Your site will be available at your-site-name.pages.dev

5. Static.app

Static.app is a simple drag-and-drop hosting service for static websites.

  • Go to static.app
  • Drag and drop your project folder or upload a ZIP file
  • Choose a subdomain for your site
  • Your site will be live immediately

Important Tips

  • Ensure all file paths in your HTML, CSS, and JavaScript files are relative
  • Test your site after deployment to ensure all features work correctly
  • If you have the OpenCalendar.bat file, update it to point to your new hosted URL
  • Keep your repository private if you have sensitive information in your code
